# Privacy Policy (Template)

Last updated: 2026-03-02

## 1. Scope

This policy applies to the WHMCS addon module **CRM Connector** ("Module").

## 2. Data Processed by the Module

The Module may process and transmit the following WHMCS client fields to your configured CRM endpoint:

- Client ID
- First name
- Last name
- Email address
- Company name
- Custom tag value

The Module also stores local sync metadata in WHMCS database tables:

- `mod_crmconnector_contacts`
- `mod_crmconnector_logs`

## 3. Purpose of Processing

Data is processed only to synchronize WHMCS clients with your CRM for sales and customer management workflows.

## 4. Legal Basis

You (the WHMCS operator/data controller) are responsible for selecting and documenting the legal basis under applicable laws (e.g. GDPR Art. 6).

## 5. Data Sharing

The Module sends data only to the CRM endpoint configured by the administrator. No default third-party endpoint is preconfigured.

## 6. Retention

Sync metadata remains in WHMCS until removed by the operator. You should define and document your retention schedule.

## 7. Security

- Use HTTPS endpoints only.
- Restrict API key access.
- Limit WHMCS admin access via role permissions.
- Keep WHMCS and server patched.

## 8. Data Subject Rights

As data controller, you must implement procedures for access, correction, deletion, and portability requests.

## 9. International Transfers

If your CRM endpoint is outside the data subject's jurisdiction, you must implement appropriate safeguards.
